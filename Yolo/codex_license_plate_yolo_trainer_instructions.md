# Codex向け指示ファイル  
## プロジェクト名
license-plate-yolo-trainer

## 目的
ローカルPC上で動作する、**静止画専用**のナンバープレート学習用アノテーション＆学習アプリを作成する。  
用途は以下の通り。

1. 学習用画像フォルダを指定する
2. 画像を1枚ずつ表示する
3. ナンバープレートに対して矩形を描画する
4. YOLO形式のラベルファイルとして保存する
5. train / val を自動分割する
6. Ultralytics YOLO で学習を開始する
7. 学習ログと学習結果保存先を確認できる

---

## 最優先方針
- **一番単純な構成**にする
- 静止画のみ対応
- クラスは **1種類固定** とする  
  - class id: `0`
  - class name: `plate`
- UI は最低限でよい
- 認証不要
- DB不要
- Docker不要
- まずはローカル実行前提
- 高機能アノテーションツールにはしない
- 回転矩形は不要
- 多クラス対応は不要
- 推論画面は必須ではないが、余力があれば簡易確認機能をつけてもよい

## 必須機能

### 1. 画像フォルダ読み込み
- ローカル上の画像フォルダを指定できること
- 対応拡張子は最低限以下
  - `.jpg`
  - `.jpeg`
  - `.png`
- 読み込んだ画像一覧をサーバ側で保持し、順番に表示できること

### 2. 画像閲覧
- 1枚ずつ表示できること
- 「前へ」「次へ」ボタンで移動できること
- 現在の画像番号 / 総枚数を表示すること
- 表示中画像のファイル名を表示すること

### 3. 矩形アノテーション
- canvas 上でドラッグして矩形を描画できること
- 1画像に複数矩形を登録できること
- 最低限以下の操作ができること
  - 矩形追加
  - 最後の矩形を削除
  - 全矩形クリア
- クラス選択UIは不要。常に `plate` 固定とすること

### 4. ラベル保存
- 保存時に YOLO形式の `.txt` を出力すること
- 形式は以下

```text
class_id x_center y_center width height
```

- 値は画像サイズ基準の正規化値とすること
- 画像ファイル名 `abc.jpg` に対して `abc.txt` を生成すること
- 既存ラベルがある場合は読み込んで再表示できるようにすること

### 5. train / val 自動分割
- 画像とラベルを保存したあと、学習開始時に dataset 配下へコピーすること
- train / val は 8:2 を基本に自動分割すること
- ランダム分割でよい
- 分割結果の件数を画面に表示すること

### 6. data.yaml 自動生成
内容は最低限以下とすること。

```yaml
path: ./workspace/dataset
train: images/train
val: images/val
names:
  0: plate
```

パスの扱いで動かない場合は、実行環境に合わせて適切に修正すること。  
ただし、**plate の単一クラス前提は維持すること**。

### 7. 学習開始
- 学習開始ボタンを設けること
- 最低限指定できるパラメータ
  - model
  - epochs
  - imgsz
  - batch
- 初期値例
  - model: `yolov8n.pt` または環境に応じた軽量モデル
  - epochs: `50`
  - imgsz: `640`
  - batch: `8`
- 学習は Python API から呼び出すこと
- 画面上に学習状態を表示すること
- 標準出力やログを簡易表示できるとよい

### 8. 学習結果表示
最低限以下を表示すること
- 学習実行の成否
- 出力ディレクトリ
- best.pt の保存先
- last.pt の保存先

## API要件
最低限、以下のような API を持つこと。  
エンドポイント名は多少変更してもよいが、責務は保つこと。

- `POST /api/load-images`
  - 画像フォルダを読み込む
- `GET /api/images`
  - 画像一覧または現在の画像情報取得
- `GET /api/image/{index}`
  - 指定画像取得
- `GET /api/annotations/{image_name}`
  - 既存アノテーション取得
- `POST /api/annotations/{image_name}`
  - アノテーション保存
- `POST /api/prepare-dataset`
  - train / val 分割と data.yaml 生成
- `POST /api/train`
  - 学習開始
- `GET /api/train-status`
  - 学習状態取得

学習処理は時間がかかるため、**バックグラウンドスレッド**または簡易ジョブ管理で実装すること。  
ただし Redis / Celery などの大がかりな仕組みは不要。

## 実装上の重要条件

### 1. YOLOラベル変換
canvas座標から元画像座標へ正しく変換すること。  
表示倍率が変わっても保存値が正しくなるようにすること。

### 2. 既存ラベル再読込
画像切り替え時に、同名 `.txt` があれば再表示すること。

### 3. 例外処理
最低限以下に対応すること。
- 指定フォルダが存在しない
- 画像が1枚もない
- 学習対象ラベルが0件
- modelファイルが見つからない
- 学習中に再度 train を押した場合

### 4. Windowsで動きやすくする
パス処理は `pathlib` を使うこと。  
Windowsでもローカル実行しやすいように実装すること。
